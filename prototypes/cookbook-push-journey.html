Push Notification Lifecycle Strategy

A behavioural lifecycle journey to drive first saves, repeat usage, and Cookbook opens across all app users. Five segments, triggered sequencing, and suppression logic. Copy aligned to Global TOV (Fresh, Vivid, Sharp, Zesty) and the Efficient Inspired audience profile.

Segment 1: Dormant 3 Pushes
Entry Trigger
User has had app installed 3+ days AND has opened app 2+ times AND has never viewed the Cookbook screen.

They're past initial app exploration but Cookbook hasn't surfaced organically. They're active enough to receive a push without it feeling spammy.

Exit Conditions
  • User opens Cookbook → graduate to Browsers
  • User saves a recipe → graduate to One and Done
  • User uninstalls app → suppress
  • 3 pushes sent with no Cookbook open → cool off 30 days, then re-enter

Segment 5: Building Habit 4 Rotating Pushes
Entry Trigger
User has saved 2+ recipes in the last 14 days OR has opened Cookbook 3+ times in the last 14 days.

Active users who are forming the habit. Don't sell the feature here. Reinforce the behaviour and introduce advanced use cases. Max 2 pushes per month.

Exit Conditions
  • No Cookbook open in 14 days → move to Lapsed Savers
  • Ongoing activity → stay in segment, rotate through content monthly
  • Cap: max 2 pushes per month for this segment

Global Rules 7 Rules
Rule Logic
Frequency Cap Max 1 Cookbook push per 5 days (across all segments)
AM Window 07:30 to 09:00 local. For discovery/save-oriented pushes. Targets the EI morning social scroll (per Efficient Inspired study: morning anchor = social media browsing).
PM Window 17:00 to 19:00 local. For "cook tonight" and action-oriented pushes. Catches the dinner decision moment when EIs finalise their spontaneous plans.
Suppression Do not send if user received any other push in the last 24 hours
Graduation Priority If a user qualifies for multiple segments, use the highest engagement segment (furthest down the funnel)
Opt-out Respect Immediate suppression on push opt-out. No fallback to other channels from this journey.
Re-entry Cooldown If a user exits and re-enters the same segment, serve fresh copy variants (not repeats)

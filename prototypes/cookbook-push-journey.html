Push Notification Lifecycle Strategy

A behavioural lifecycle journey to drive first saves, repeat usage, and Cookbook opens across all app users. Five segments, triggered sequencing, and suppression logic.

Segment Definition Goal

Browsers Opened Cookbook, never saved a recipe First save

Lapsed Savers Saved 2+ recipes, no Cookbook open in 14+ days Re-open
Building Habit Saved 2+ recipes in the last 14 days Increase frequency
Segment 1: Dormant 3 Pushes
Entry Trigger
User has had app installed 3+ days AND has opened app 2+ times AND has never viewed the Cookbook screen.

They're past initial app exploration but Cookbook hasn't surfaced organically. They're active enough to receive a push without it feeling spammy.

Exit Conditions
  • User opens Cookbook → graduate to Browsers
  • User saves a recipe → graduate to One and Done
  • User uninstalls app → suppress
  • 3 pushes sent with no Cookbook open → cool off 30 days, then re-enter

Segment 3: One and Done 3 Pushes
Entry Trigger
User has exactly 1 saved recipe AND it has been 7+ days since that save AND no new Cookbook activity in that window.

Gives them natural time to come back organically. If they don't after 7 days, the habit hasn't formed and they need a nudge.

Exit Conditions
  • User saves a second recipe → graduate to Building Habit
  • No second save after 3 pushes → cool off 21 days, re-enter with different angle
  • User opens Cookbook but doesn't save → send Browser-style friction-reducing push

Segment 4: Lapsed Savers 3 Pushes
Entry Trigger
User has 2+ saved recipes AND last Cookbook open was 14+ days ago AND user has opened the app at least once in the last 7 days (still active, just not using Cookbook).

If they've gone dark on the whole app, this is a broader retention problem. We only nudge people who are reachable and still using HelloFresh.

Exit Conditions
  • User opens Cookbook → graduate back to Building Habit
  • User saves a new recipe → graduate to Building Habit
  • No re-engagement after 3 pushes → cool off 30 days
  • User goes inactive on app entirely → exit to general retention journey

Segment 5: Building Habit 4 Rotating Pushes
Entry Trigger
User has saved 2+ recipes in the last 14 days OR has opened Cookbook 3+ times in the last 14 days.

Active users who are forming the habit. Don't sell the feature here. Reinforce the behaviour and introduce advanced use cases. Max 2 pushes per month.

Exit Conditions
  • No Cookbook open in 14 days → move to Lapsed Savers
  • Ongoing activity → stay in segment, rotate through content monthly
  • Cap: max 2 pushes per month for this segment

Global Rules 6 Rules
Rule Logic
Frequency Cap Max 1 Cookbook push per 5 days (across all segments)
Send Window 11:00 to 13:00 local OR 17:00 to 19:00 local (meal planning windows)
Suppression Do not send if user received any other push in the last 24 hours
Graduation Priority If a user qualifies for multiple segments, use the highest engagement segment (furthest down the funnel)
Opt-out Respect Immediate suppression on push opt-out. No fallback to other channels from this journey.
Re-entry Cooldown If a user exits and re-enters the same segment, serve fresh copy variants (not repeats)
